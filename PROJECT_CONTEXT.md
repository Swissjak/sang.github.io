## Важные технические детали

- Файлы `.txt` читаются корректно как UTF-8.
- При открытии сайта через `file://` браузер может блокировать загрузку `.txt`.
- Для нормальной работы лучше запускать сайт через локальный сервер или публиковать онлайн.
- Лучший бесплатный формат для проекта: статический сайт на `HTML + CSS + JS`.

## Как устроены тесты сейчас

- В интерфейсе используются аккуратные названия тем, а не имена файлов вроде `УК.txt`.
- Для темы нужен отдельный ручной файл вопросов формата `...Вопросы.txt`.
- Поддерживается новый формат файлов вопросов вида:
  номер статьи на отдельной строке, затем строка `Вопрос: ...`, затем варианты `A) ...` и строка `Ответ: X`.
- В ручных вопросах варианты ответов тоже перемешиваются автоматически.
- Билеты не фиксированные: вопросы выбираются в случайном порядке.
- Для всех тем предусмотрен единый формат отдельных файлов вопросов `...Вопросы.txt`.
- Если отдельного файла вопросов нет, тема считается неготовой к экзамену.
- Логика тестирования находится в [app.js](C:\Users\core\Desktop\SANG SITE\app.js).
- Для локального запуска добавлены [server.js](C:\Users\core\Desktop\SANG SITE\server.js) и [start-server.bat](C:\Users\core\Desktop\SANG SITE\start-server.bat).
